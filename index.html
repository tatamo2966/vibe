// WHAT IS VIBE CODING

코드보다
의도가 먼저

바이브 코딩은 Andrej Karpathy가 제시한 개념으로, AI에게 구현의 세부사항을 맡기고 개발자는 무엇을 만들지, 왜 만드는지에 집중하는 방식입니다.

기존 개발이 "어떻게 만드느냐"에 에너지를 썼다면, 바이브 코딩은 "무엇을 만드느냐"에 집중합니다. 문법보다 사고, 타이핑보다 판단, 구현보다 설계가 핵심 역량이 됩니다.

비개발자도 아이디어를 즉시 코드로 만들 수 있고, 숙련 개발자는 생산성을 몇 배로 높일 수 있습니다.

// KEY FEATURES

왜 바이브 코딩인가

전통적인 개발 방식과 비교했을 때 바이브 코딩이 가져오는 변화

→ 01
진입 장벽 붕괴
문법을 몰라도, 프레임워크를 처음 봐도 괜찮습니다. 아이디어가 있고 의도를 설명할 수 있다면 누구든 프로덕트를 만들 수 있습니다.
→ 02
10배 빠른 프로토타이핑
아이디어에서 동작하는 코드까지 걸리는 시간을 극적으로 단축합니다. 실험하고, 확인하고, 방향을 바꾸는 사이클이 몇 시간에서 몇 분으로.
→ 03
컨텍스트 중심 사고
AI에게 올바른 컨텍스트를 주는 능력이 핵심 역량이 됩니다. 프롬프트 설계, 요구사항 정의, 결과물 검토라는 새로운 스킬셋이 중요해집니다.
→ 04
흐름 상태 유지
구현 세부사항에서 벗어나 더 큰 그림에 집중할 수 있습니다. 창의적인 흐름이 끊기지 않고 계속 이어집니다.
→ 05
AI와 협업하는 방식
AI를 도구로 쓰는 것이 아니라 페어 프로그래머로 대하는 마인드셋. 대화하고, 검토하고, 방향을 제시하는 역할이 개발자에게 남습니다.
→ 06
검증과 판단 역량
AI가 만든 코드를 이해하고 검증하는 능력은 여전히 중요합니다. 맹목적으로 수용하지 않고 비판적으로 검토하는 사고방식이 필요합니다.

바이브 코딩
시작하는 법

복잡하지 않습니다. 다음 흐름만 기억하면 됩니다.

01

의도를 명확하게 정의한다

무엇을 만들고 싶은지, 사용자는 누구인지, 어떤 문제를 해결하는지를 먼저 정리하세요. AI에게 전달하는 컨텍스트의 품질이 결과물의 품질을 결정합니다.

// CONTEXT FIRST
02

AI와 대화하듯 요청한다

기술적인 명세서를 쓸 필요 없습니다. 동료에게 설명하듯 자연스럽게 이야기하세요. "사용자가 이메일로 가입하고 로그인할 수 있게 해줘. 비밀번호 찾기도 포함해서."

// NATURAL LANGUAGE
03

결과물을 검토하고 방향을 조정한다

AI가 만든 코드를 실행해보고 의도와 맞는지 확인하세요. 맞지 않는 부분은 구체적으로 피드백합니다. "결제 성공 후 이메일 알림이 빠졌어. 추가해줘."

// ITERATE & REVIEW
04

흐름을 유지하며 계속 나아간다

세부 구현에 막히지 말고 계속 앞으로 나아가세요. 완성된 기능을 토대로 다음 기능을 요청하고, 프로덕트를 점진적으로 완성해갑니다.
